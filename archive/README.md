\# Development Archives \& Experimental Logs



\## Overview

このディレクトリは、本プロジェクト（AI-Driven Trend Following System）の完成に至るまでの、\*\*全ての仮説検証と失敗の記録\*\*を保管するアーカイブです。



ここにあるコード群は、最終的なシステム（`src/`）には採用されませんでしたが、\*\*「なぜその手法が機能しないのか（Why it failed）」\*\*を理解し、現在のロジックを導き出すために不可欠な実験データです。



\## Directory Structure

各フェーズにおける検証内容は以下の通りです。



\### 📂 00\_data\_analyze

\* \*\*検証内容\*\*: GBDTにおける特徴量の効果確認と設定(meta\_short、meta\_longの発見)





\### 📂 01\_rl\_experiments (Reinforcement Learning)

\* \*\*検証内容\*\*: 強化学習（PPO/DQN）を用いた自律売買エージェントの開発。

\* \*\*棄却理由\*\*: 金融市場の非定常性（Non-stationarity）により、エージェントが安定した方策を学習できず、過学習または無行動に陥ったため。



\### 📂 02\_nlp\_finbert (Sentiment Analysis)

\* \*\*検証内容\*\*: 金融特化BERT（FinBERT）を用いたニュース見出しのセンチメント分析と、市場方向性の予測。

\* \*\*棄却理由\*\*: 情報の即時織り込み（Efficient Market）により、ニュース発表後の優位性が確認できず、予測精度がランダム（~50%）に収束したため。



\### 📂 03\_market\_analysis\_mixed (GBDT \& Scalping)

\* \*\*検証内容\*\*: LightGBM等を用いた、日本株および暗号資産（Bitcoin）の高頻度取引（Scalping）検証。

\* \*\*棄却理由\*\*:

&nbsp;   \* \*\*手数料負け\*\*: スプレッドと取引手数料が、短期予測の期待値を上回る。

&nbsp;   \* \*\*逆選択\*\*: 指値注文において、不利な価格変動時のみ約定する「落ちるナイフ」現象を確認。
